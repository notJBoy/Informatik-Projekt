## Neuer Eintrag: Spracheinstellungen (18.03.2026)

- Prompt:
  - "Füge in den Account-Einstellungen ganz oben eine option zum Einstellen der Sprache hinzu. Diese sollte für den User über das backend in der Datenbank gespeichert werden und beim aufrufen der Seite dementsprechend richtig ausgewählt sein. Benutze die gängigsten europäischen Sprachen."

- Umsetzung:
  1. Datenbankfeld `language` zur `users`-Tabelle in `backend.py` hinzugefügt.
  2. Migrationspfad für existierende DBs hinzugefügt: `ALTER TABLE users ADD COLUMN language TEXT DEFAULT 'de'`.
  3. Registrierungs-endpoint in `backend.py` ergänzt, um neue User mit Standardwert `de` anzulegen.
  4. Login-Endpoint gibt `language` zurück.
  5. Neuer Endpoint `GET /auth/user/{user_id}` implementiert.
  6. Neuer Endpoint `PUT /auth/change-language/{user_id}` implementiert.
  7. In `current_dashboard.php`: neue Spracheinstellung im Account Modal, plus `loadAccountSettings()` und `changeLanguage()` JS-Funktionen.
  8. In `auth/login.php`: Session-Feld `language` beim Login gesetzt.

- Ergebnis:
  - Sprache wird in DB persistiert.
  - Spracheinstellung in Account Modal wird beim Öffnen geladen und vorausgewählt.
  - Sprache kann gehändelt werden (de, en, fr, es, it, nl, pl, sv).
  - Backend ohne Syntaxfehler (py_compile OK).
  - API-Verbindungskonfiguration entkoppelt von Festwerten in `api_config.php`.
  - `auth/login.php`, `auth/register.php` und `current_dashboard.php` nutzen `API_BASE_URL` statt `http://localhost:8000`.
  - Server-Endpunkte können zentral umgestellt werden.
